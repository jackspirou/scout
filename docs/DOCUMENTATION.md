## Canonical set

Only these documents are authoritative:

- `README.md` — Project overview, installation, usage
- `AGENTS.md` — Build commands, architecture, AppKit patterns, common mistakes (`CLAUDE.md` is a symlink)
- `CHANGELOG.md` — Release history (auto-generated by git-cliff)
- `docs/DOCUMENTATION.md` — How to document and find documentation
- `docs/scout-prd.md` — Product requirements document
- `docs/scoutdrop.md` — ScoutDrop peer-to-peer file transfer protocol and architecture

Anything else in `docs/archive/` is historical context only.

## Required metadata

Every canonical doc (except `CHANGELOG.md`, which is auto-generated) must start with:

- `Status` — `Canonical`, `Draft`, or `Archived`
- `Owner` — Team or person responsible
- `Last verified` — Date someone confirmed the content is accurate
- `Source of truth` — Real source files, not vague prose

## Update policy

When architecture, build process, or public interfaces change:

1. Update the source code
2. Update the affected canonical doc in the same change
3. Archive or delete superseded docs instead of leaving two competing explanations

Move outdated docs to `docs/archive/` with `Status: Archived`.

## Adding a new doc

1. Create it in `docs/` with the required metadata header
2. Add it to the canonical set list above
3. Commit both changes together
